Web Application Forensics & Linux FirewallsProject OverviewThis project focuses on the fundamentals of Digital Forensics and Incident Response (DFIR), specifically targeting web applications and Linux system security. It covers the analysis of system logs, network traffic, and the implementation of host-based firewalls like firewalld and iptables.The goal is to develop the ability to trace the origin of cyber attacks, identify vulnerabilities, and document findings in a legally sound and professional forensic report.Learning ObjectivesBy the end of this project, I can explain:Digital Forensics: The process of uncovering and interpreting electronic data.Web Application Forensics: Core concepts of investigating attacks targeting the web layer.Log Analysis: How to use auth.log, access_log, and dmesg to reconstruct an attacker's timeline.Tool Proficiency: Using Wireshark for packet analysis and Burp Suite for intercepting web traffic.Legal Frameworks: Best practices and legal standards for maintaining the chain of custody.RequirementsEnvironment: All scripts are designed and tested on Kali Linux.Editors: Developed using vi, vim, or emacs.Script Standards:First line is exactly #!/bin/bash.All files end with a new line.No usage of backticks, &&, ||, or ;.Scripts are compliant with the Betty style guide.All files are executable.File DescriptionsFileDescriptionauth.logContains system authorization information, including user logins and authentication machinery.dmesgUsed to examine or control the kernel ring buffer; vital for detecting hardware-level or kernel-based anomalies.README.mdThis documentation file.Forensic Investigation ResourcesLog Files used in Analysis:auth.log: Crucial for identifying brute-force attacks and unauthorized SSH attempts.dmesg: Useful for finding traces of rootkits or hardware-level tampering during an investigation.Networking & Firewalls:firewalld vs iptables: Understanding the transition from the legacy iptables rulesets to the dynamic zone-based management of firewalld.Network Traffic: Utilizing Wireshark to identify SQL injection or XSS patterns within PCAP files.How to Run ScriptsEnsure the script has executable permissions:Bashchmod +x <script_name>
